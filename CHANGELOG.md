# Changelog

## 4.4.0

- 将原有整体透明度拆分为文字透明度与背景透明度，两者可独立设置。
- 背景透明度范围扩展为 0.00～1.00，设置为 0 时完全移除背景填充。
- Windows 字幕渲染从 Tk 颜色键透明升级为 `UpdateLayeredWindow` 每像素 Alpha 合成。
- 新渲染器使用 Pillow 输出抗锯齿文字，消除透明模式下的黑边、硬重影和脏色。
- 无背景模式自动隐藏状态栏、居中排版，并使用柔化阴影与轻描边增强复杂画面可读性。
- 定位模式在全透明配置下临时显示淡色拖动区域，确保整个字幕框可见且可拖动。
- 新增 `native_popup_text_opacity` 与 `native_popup_background_opacity` 配置字段。
- 保留旧版 `native_popup_opacity` 和 `native_popup_transparent_background` 的兼容映射。
- 配置架构升级到 v4；旧版整体透明度迁移为背景透明度，文字默认保持完全不透明。
- 新增每像素 Alpha、透明文字、独立透明度和中文换行回归测试。

## 4.3.0

- 新增“无背景填充”模式，Windows 下使用颜色键透明，仅保留字幕文字。
- 透明背景在窗口首次绘制后再启用，避免与 4.2.1 的安全显示和穿透时序冲突。
- 新增透明模式文字阴影开关、阴影颜色和 1～8 像素偏移设置。
- 透明模式自动隐藏字幕边框并缩小容器内边距，保留状态文字、流式更新和单窗口复用。
- 字幕设置页预览增加透明棋盘背景和文字阴影效果。
- `/health` 增加透明背景状态，独立字幕配置接口支持全部新增字段。
- 配置架构升级到 v3，旧版配置自动补齐新字段，默认仍使用背景填充。
- 新增透明背景配置、阴影参数、接口字段和静态页面回归测试。

## 4.2.1

- 修复启用鼠标穿透后字幕窗口可能完全不显示的问题。
- 调整 Windows 样式应用顺序：字幕先完成首次显示和绘制，再延迟启用穿透。
- 穿透切换只修改 `WS_EX_TRANSPARENT`，不再重复修改 Tk 的分层窗口样式。
- 为流式正文与推理更新增加原生窗口重绘，确保穿透状态下持续显示增量内容。
- 定位模式在字幕窗口创建前即强制关闭穿透，预览始终可见且可拖动。
- 自定义坐标超出当前虚拟桌面时自动收敛到可见区域，避免找不到字幕。
- 鼠标穿透默认改为关闭；从 4.2.0 升级时执行一次安全迁移，恢复可见字幕。
- 新增推理分片创建字幕、定位初始状态和 4.2.0 配置迁移回归测试。

## 4.2.0

- 新增可配置的字幕鼠标穿透模式，默认开启。
- Windows 下通过 `WS_EX_TRANSPARENT` 让点击、滚轮和拖动直接作用于字幕下方窗口。
- 新增定位模式；进入后临时关闭穿透，允许拖动字幕。
- 拖动松开并保存坐标后，自动退出定位模式并恢复用户配置的穿透状态。
- 定位模式最长持续 60 秒，超时自动恢复穿透，避免字幕长期拦截鼠标。
- 新增 `/admin/subtitle-positioning/start` 与 `/admin/subtitle-positioning/finish` 接口。
- 字幕设置页新增鼠标穿透开关、“进入定位模式”和“结束定位”操作。
- `/health` 新增 `native_popup_click_through` 状态。
- 新增穿透配置、定位模式切换与坐标保存恢复测试。

## 4.1.0

- 将字幕配置从“转发配置”迁移到独立“字幕设置”选项卡。
- 新增 `/admin/subtitle-config` 独立读取与保存接口。
- 新增字幕背景、正文、辅助文字、边框和错误提示颜色配置。
- 新增字幕拖动定位；松开鼠标后通过 IPC 自动保存绝对坐标。
- 新增 `custom` 自定义位置以及跨多显示器的负坐标支持。
- 原生字幕改为全局单实例；新请求清空并复用当前窗口。
- 新请求获得显示权后，丢弃被替代请求的后续字幕事件，避免并发串台。
- 新增单实例复用、位置持久化、字幕接口和颜色校验测试。

## 4.0.0

- 将 1388 行的单体 `app.py` 重构为 `llm_relay_desk` 分层包。
- 按职责拆分系统、管理、监视、OpenAI 和 Ollama 路由。
- 将配置校验、JSON 存储、提示词服务、事件监视、流解析和上游转发解耦。
- 使用 `Runtime` 聚合运行期依赖，路由不再依赖模块级业务全局变量。
- 使用 FastAPI lifespan 统一管理原生字幕进程启动与关闭。
- 保留原有 API 路径、配置文件格式、静态页面和启动脚本。
- 根目录 `popup_window.py` 调整为兼容导入层，实际实现迁移到 `desktop/window.py`。
- 新增架构文档、`pyproject.toml`、开发依赖和自动化测试。
- 新增 OpenAI SSE 与 Ollama NDJSON 端到端回归测试。

## 3.2.0

- 将原生响应弹窗重构为无标题栏桌面字幕浮层。
- 新增九宫格固定位置和 X/Y 像素偏移设置。
- 新增字幕宽高、字号、透明度和推理显示开关。
- 新增“预览流式字幕”管理接口和 UI 按钮。
- 字幕窗口保持置顶，但在 Windows 下不主动激活或抢占输入焦点。
- API 测试页默认发送 `stream: true`，通过浏览器 `ReadableStream` 实时解析 SSE。
- 测试页最终回答和完整响应在生成过程中持续更新。
- 明确区分真实流式请求与 `stream: false` 的一次性完整响应。

## 3.1.0

- 新增独立进程运行的原生桌面响应弹窗。
- 新增管理页“原生响应弹窗”开关。
- 新增完成后自动关闭时间设置，默认 30 秒，范围 1～3600 秒。
- 流式正文与推理内容在原生窗口中分离显示。
- 多个并发请求分别创建窗口。
- Windows 下使用非激活显示方式，避免主动抢占输入焦点。
- 关闭原生弹窗开关时同步关闭当前弹窗。
- 原生弹窗进程、队列或桌面环境异常不会阻塞 API。
- `启动WebUI.bat` 不再自动打开 Web 监视器。

## 3.0.0

- 项目名称调整为 **LLM Relay Desk**。
- 新增独立实时响应页 `/monitor/`。
- 新增 WebSocket 事件通道 `/ws/monitor`。
- 新增 OpenAI SSE 正文与推理内容解析。
- 新增 Ollama NDJSON 正文与思考内容解析。
- 新增非流式响应捕获和前端平滑流式展示。
- 新增多请求分流、请求状态、耗时、来源及请求 ID 展示。
- 新增最近 60 条内存历史和清空接口。
- 监视事件采用非阻塞有界队列，慢客户端不会阻塞 API。
- API 响应新增 `X-Relay-Request-ID` 响应头。
- 新增 Windows 独立监视窗口启动脚本。
