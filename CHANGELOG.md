# Changelog

## 4.1.0

- 将字幕配置从“转发配置”迁移到独立“字幕设置”选项卡。
- 新增 `/admin/subtitle-config` 独立读取与保存接口。
- 新增字幕背景、正文、辅助文字、边框和错误提示颜色配置。
- 新增字幕拖动定位；松开鼠标后通过 IPC 自动保存绝对坐标。
- 新增 `custom` 自定义位置以及跨多显示器的负坐标支持。
- 原生字幕改为全局单实例；新请求清空并复用当前窗口。
- 新请求获得显示权后，丢弃被替代请求的后续字幕事件，避免并发串台。
- 新增单实例复用、位置持久化、字幕接口和颜色校验测试。

## 4.0.0

- 将 1388 行的单体 `app.py` 重构为 `llm_relay_desk` 分层包。
- 按职责拆分系统、管理、监视、OpenAI 和 Ollama 路由。
- 将配置校验、JSON 存储、提示词服务、事件监视、流解析和上游转发解耦。
- 使用 `Runtime` 聚合运行期依赖，路由不再依赖模块级业务全局变量。
- 使用 FastAPI lifespan 统一管理原生字幕进程启动与关闭。
- 保留原有 API 路径、配置文件格式、静态页面和启动脚本。
- 根目录 `popup_window.py` 调整为兼容导入层，实际实现迁移到 `desktop/window.py`。
- 新增架构文档、`pyproject.toml`、开发依赖和自动化测试。
- 新增 OpenAI SSE 与 Ollama NDJSON 端到端回归测试。

## 3.2.0

- 将原生响应弹窗重构为无标题栏桌面字幕浮层。
- 新增九宫格固定位置和 X/Y 像素偏移设置。
- 新增字幕宽高、字号、透明度和推理显示开关。
- 新增“预览流式字幕”管理接口和 UI 按钮。
- 字幕窗口保持置顶，但在 Windows 下不主动激活或抢占输入焦点。
- API 测试页默认发送 `stream: true`，通过浏览器 `ReadableStream` 实时解析 SSE。
- 测试页最终回答和完整响应在生成过程中持续更新。
- 明确区分真实流式请求与 `stream: false` 的一次性完整响应。

## 3.1.0

- 新增独立进程运行的原生桌面响应弹窗。
- 新增管理页“原生响应弹窗”开关。
- 新增完成后自动关闭时间设置，默认 30 秒，范围 1～3600 秒。
- 流式正文与推理内容在原生窗口中分离显示。
- 多个并发请求分别创建窗口。
- Windows 下使用非激活显示方式，避免主动抢占输入焦点。
- 关闭原生弹窗开关时同步关闭当前弹窗。
- 原生弹窗进程、队列或桌面环境异常不会阻塞 API。
- `启动WebUI.bat` 不再自动打开 Web 监视器。

## 3.0.0

- 项目名称调整为 **LLM Relay Desk**。
- 新增独立实时响应页 `/monitor/`。
- 新增 WebSocket 事件通道 `/ws/monitor`。
- 新增 OpenAI SSE 正文与推理内容解析。
- 新增 Ollama NDJSON 正文与思考内容解析。
- 新增非流式响应捕获和前端平滑流式展示。
- 新增多请求分流、请求状态、耗时、来源及请求 ID 展示。
- 新增最近 60 条内存历史和清空接口。
- 监视事件采用非阻塞有界队列，慢客户端不会阻塞 API。
- API 响应新增 `X-Relay-Request-ID` 响应头。
- 新增 Windows 独立监视窗口启动脚本。
